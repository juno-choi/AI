### AI와 협업
각자의 역할로 분리한다.
예를 들어 claude는 워커, codex는 팀장, gemini는 검증 및 간단한 코드 작성 or 프론트
각 ai들의 특징에 맞춰 역할을 부여한다.

claude 워커
- opus 계획을 잘 세움
- sonnet 코드 작성 잘함
- skills와 agent를 활용하여 커스텀 가능
- 업무가 종료될 때마다 clear를 통해 context 비움

gpt 팀장
- 논리적인 부분에서 우수함(전체 맥락 파악)
- claude가 세운 계획을 검증하는 역할
- 업무 context를 가져가고 있어야 함.

gemini 워커
- 프론트쪽 역량이 좋음
- 간단한 수정 업무는 gemini를 통해 하는 것이 빠름
- claude가 해결하지 못하는 문제를 gemini가 해결하는 경우가 있음

역할에 맞춰서 업무를 진행한다.

하지만, 모든 업무를 ai를 믿고 진행하면 안된다. 하드 리셋 하고 싶지 않으면
또한 모든 수정 사항은 깃으로 커밋하며 남기고 테스트 코드를 통해 진행하는 업무를 검증한다.

### 업무 흐름
1. 내가 하고자 하는 업무에 대해 계획을 세운다.
2. 계획을 claude에게 먼저 검토한다.
3. claude가 생성한 계획을 codex에게 검토 시킨다.
4. codex와 수정한 계획을 다시 claude에게 검토 시킨다.
5. 검토가 완료된 계획을 phase 별로 나눈다.
6. 나눈 업무에 대해 codex에게 phase 별로 업무지시를 어떻게 하면 좋을지 질문한다.
7. 해당 프롬프트를 워커에게 실행하도록 한다.
